The OI Resume App uses **XCUITest**, Apple's native UI testing framework, to ensure the reliability and quality of the SwiftUI application. This testing suite validates user interactions, UI element visibility, and navigation flows.

## Testing architecture

The test implementation follows industry best practices:

- **XCUITest Framework**: Apple's native UI testing framework for iOS applications
- **Page Object Model (POM)**: Encapsulates UI elements and actions into reusable page objects
- **Base Test Class**: Provides common setup and teardown logic for all tests
- **Modular Structure**: Separates test logic, page objects, and base configuration

## Project structure

## Base test class

The `OIBaseTest` class provides common functionality for all UI tests:

```swift OI_PortfolioUITests/OIBaseTest/OIBase.swift
import Foundation
import XCTest

class OIBaseTest: XCTestCase {
    let app = XCUIApplication()
    
    override func setUpWithError() throws {
        super.setUp()
        continueAfterFailure = false
        app.launch()
        
    }

    override func tearDownWithError() throws {
        super.tearDown()
        app.terminate()
        
    }
}
```

### Key features

- **XCUIApplication Instance**: Shared app instance available to all test subclasses
- **Setup**: Launches the app before each test and disables continuation after failures
- **Teardown**: Terminates the app after each test to ensure clean state
- **Inheritance**: All test classes inherit from `OIBaseTest` for consistent behavior

<Note>
Setting `continueAfterFailure = false` ensures that tests stop immediately when an assertion fails, making it easier to identify and debug issues.
</Note>

## Benefits of XCUITest

<AccordionGroup>
  <Accordion title="Native integration">
    XCUITest is built by Apple and integrates seamlessly with Xcode, Swift, and iOS development workflows.
  </Accordion>
  
  <Accordion title="Real device testing">
    Tests can run on simulators and real devices, ensuring compatibility across different iOS versions and device types.
  </Accordion>
  
  <Accordion title="Accessibility-first">
    XCUITest leverages accessibility identifiers and labels, which improves both testability and app accessibility.
  </Accordion>
  
  <Accordion title="Swift language support">
    Tests are written in Swift, allowing developers to use the same language for both app code and tests.
  </Accordion>
</AccordionGroup>
