## Build and run

Get the OI Resume App running on your iOS simulator in just a few steps.

<Steps>
  <Step title="Clone the repository">
    Clone the repository and navigate to the project directory:
    
    ```bash
    git clone https://github.com/olaeben/Resume-App.git
    cd Resume-App
    ```
  </Step>

  <Step title="Open in Xcode">
    Open the Xcode project:
    
    ```bash
    open OI_Portfolio.xcodeproj
    ```
  </Step>

  <Step title="Select a simulator">
    In Xcode's toolbar, select a target device from the simulator menu. We recommend:
    - iPhone 14 Pro
    - iPhone 15
    - Any device running iOS 14.0 or later
  </Step>

  <Step title="Build and run">
    Click the Run button (▶️) or press `Cmd + R` to build and run the app.
    
    The app will compile and launch in the iOS Simulator.
  </Step>
</Steps>

<Note>
  The first build may take a few minutes as Xcode compiles all the source files and resources.
</Note>

### Step-by-step customization

<Steps>
  <Step title="Open AppModel.swift">
    In Xcode's Project Navigator, navigate to:
    ```
    OI_Portfolio/Code/Services/AppModel.swift
    ```
  </Step>

  <Step title="Update basic information">
    Replace the portfolio properties with your information:
    
    ```swift
    var portfolio: Portfolio = Portfolio(
        name: "Your Name",
        role: "Your Role",
        description: "Your professional description.",
        linkedin: "your-linkedin-username",
        github: "your-github-username",
        location: "Your Location",
        // ...
    )
    ```
  </Step>

  <Step title="Customize skills">
    Modify the skills array with your technical skills. Use SF Symbols for icons:
    
    ```swift
    skills: [
        Skill(id: UUID().uuidString, skillName: "Swift", image: "swift"),
        Skill(id: UUID().uuidString, skillName: "React", image: "atom"),
        Skill(id: UUID().uuidString, skillName: "Python", image: "terminal"),
        // Add more skills...
    ]
    ```
    
    <Note>
      Browse available SF Symbols at [Apple's SF Symbols app](https://developer.apple.com/sf-symbols/) or use the SF Symbols app in macOS.
    </Note>
  </Step>

  <Step title="Update experiences">
    Add your professional experience:
    
    ```swift
    experience: [
        Experience(
            id: UUID().uuidString,
            companyName: "Your Company",
            role: "Your Role",
            duration: "Start Date - End Date"
        ),
        // Add more experiences...
    ]
    ```
  </Step>

  <Step title="Add your profile photo">
    Replace the profile photo in Assets.xcassets:
    
    1. In Xcode, navigate to `OI_Portfolio/Assets.xcassets`
    2. Find the `display_photo` image set
    3. Drag your profile photo (recommended: 300x300px) into the image well
    4. Ensure it's set for "Universal" deployment
  </Step>
</Steps>

## Understanding the code structure

Let's explore the key components:

### Data models

The app uses three main models defined in `Portfolio.swift`:

<CodeGroup>
```swift OI_Portfolio/Code/Models/Portfolio.swift
struct Skill: Identifiable {
    var id: String
    let skillName: String
    let image: String  // SF Symbol name
}

struct Experience: Identifiable {
    var id: String
    let companyName: String
    let role: String
    let duration: String
}

struct Portfolio {
    let name: String
    let role: String
    let description: String
    let linkedin: String
    let github: String
    let location: String
    let skills: [Skill]
    let experience: [Experience]
}
```
</CodeGroup>

### Main view structure

The `PortfolioView` is the main container that composes all sections:

<CodeGroup>
```swift OI_Portfolio/Code/Views/PortfolioView.swift
struct PortfolioView: View {
    var appModel: AppModel = AppModel()
    
    var body: some View {
        ZStack {
            Color(UIColor.systemBackground)

                    .padding(.top, 32)
                    
                    ExperiencesView(
                        experience: appModel.portfolio.experience
                    )
                    .padding(.top, 42)
                }
                .padding(24)
            }
        }
    }
}
```
</CodeGroup>

### Collapsible sections

Both `SkillsView` and `ExperiencesView` implement collapsible functionality:

<CodeGroup>
```swift OI_Portfolio/Code/Views/SupportViews/SkillsView.swift
struct SkillsView: View {
    var skills: [Skill]
    var width: CGFloat
    
    @State var showSkills = true
    
    var body: some View {
        VStack(alignment: .leading) {
            HStack(spacing: 16) {
                Text("Skills")
                    .font(Montserrat.bold.font(size: 21))
                    .opacity(0.9)
                
                Button {
                    withAnimation(.easeInOut(duration: 0.4)) {
                        showSkills.toggle()
                    }
                } label: {
                    Image(systemName: "chevron.up")
                        .font(.system(size: 18, weight: .medium))
                        .rotationEffect(showSkills ? .zero : .degrees(180))
                }
                .buttonStyle(PlainButtonStyle())
            }
            
            if showSkills {
                LazyVGrid(
                    columns: [GridItem(), GridItem(), GridItem()],
                    alignment: .leading,
                    spacing: 12
                ) {
                    ForEach(skills) { skill in
                        SkillView(skill: skill, width: width / 3 - 24)
                    }
                }
                .padding(.top, 38)
            }
        }
    }
}
```
</CodeGroup>

<Note>
  The `@State` property wrapper makes the toggle reactive, and `withAnimation` provides smooth transitions.
</Note>

## Run tests

The app includes comprehensive XCUITests using the Page Object Model pattern.

### Execute UI tests

<Steps>
  <Step title="Open test navigator">
    In Xcode, press `Cmd + 6` to open the Test Navigator.
  </Step>

  <Step title="Run all tests">
    Click the play button next to "OI_PortfolioUITests" to run all UI tests.
    
    Or press `Cmd + U` to run all tests in the project.
  </Step>

  <Step title="View test results">
    Watch as the simulator automatically:
    - Launches the app
    - Verifies the author's name exists
    - Toggles skills section on and off
    - Toggles experience section on and off
    - Confirms all elements are visible
  </Step>
</Steps>

### Test structure

The main UI test verifies all interactive features:

<CodeGroup>
```swift OI_PortfolioUITests/OITest/OI_PortfolioUITests.swift
final class OI_ResumeTests: OIBaseTest {
    func testResumePage() throws {
        // Initialize the Page Object
        let resumePage = ResumePage(app: app)
        
        // Verify Author's Name
        resumePage.verifyAuthorNameExists()
        
        // Verify and Toggle Skills Section
        resumePage.verifySkillsVisibility(shouldExist: true)
        resumePage.toggleSkillsSection()
        resumePage.verifySkillsVisibility(shouldExist: false)
        
        // Verify and Toggle Experience Section
        resumePage.verifyExperienceVisibility(shouldExist: true)
        resumePage.toggleExperienceSection()
        resumePage.verifyExperienceVisibility(shouldExist: false)
        
        // Toggle Back the Skills and Experience Sections
        resumePage.toggleExperienceSection()
        resumePage.toggleSkillsSection()
    }
}
```
</CodeGroup>

<Note>
  The Page Object Model pattern separates test logic from page interactions, making tests easier to maintain.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="App crashes on launch">
    If the app crashes immediately:
    
    1. Check the Xcode console for error messages
    2. Verify all assets (fonts, images) are properly included
    3. Clean build folder: Product → Clean Build Folder (`Cmd + Shift + K`)
    4. Reset simulator: Device → Erase All Content and Settings
  </Accordion>

  <Accordion title="Custom fonts not appearing">
    If your custom fonts don't render:
    
    1. Verify font files are added to the project target
    2. Check `OI-Portfolio-Info.plist` includes font names in `UIAppFonts`
    3. Use the exact font name (case-sensitive)
    4. Rebuild the project
  </Accordion>

  <Accordion title="Profile photo not showing">
    If your profile photo doesn't appear:
    
    1. Ensure the image is named `display_photo` in Assets.xcassets
    2. Check that the image set has a Universal target
    3. Verify the image file is not corrupted
    4. Recommended size: 300x300px or larger
  </Accordion>

  <Accordion title="SF Symbols not rendering">
    If skill icons don't show:
    
    1. Verify you're using valid SF Symbol names
    2. Browse available symbols in the SF Symbols app
    3. Some symbols require iOS 15+ (check compatibility)
    4. Use fallback symbols for older iOS versions
  </Accordion>
</AccordionGroup>
