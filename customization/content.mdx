All portfolio content is managed through the `AppModel` class, which provides a centralized data source for your personal information, skills, and work experience.

## AppModel overview

The `AppModel` class in `Code/Services/AppModel.swift` is an `ObservableObject` that holds your portfolio data. This architecture allows views to automatically update when data changes.

```swift
class AppModel: ObservableObject {
    var portfolio: Portfolio = Portfolio(
        // ... your portfolio data
    )
}
```

## Portfolio data structure

The `Portfolio` struct from `Code/Models/Portfolio.swift` defines the content structure:

```swift
struct Portfolio {
    let name: String
    let role: String
    let description: String
    let linkedin: String
    let github: String
    let location: String
    let skills: [Skill]
    let experience: [Experience]
}
```

## Updating personal information

<Steps>
  <Step title="Open AppModel.swift">
    Navigate to `Code/Services/AppModel.swift` in your Xcode project.
  </Step>

  <Step title="Modify the Portfolio initializer">
    Update the personal information fields:
    
    ```swift
    var portfolio: Portfolio = Portfolio(
        name: "Your Name",
        role: "Your Job Title",
        description: "Your professional summary.",
        linkedin: "your-linkedin-username",
        github: "your-github-username",
        location: "Your Location",
        skills: [...],
        experience: [...]
    )
    ```
  </Step>

  <Step title="Save and rebuild">
    Save the file and rebuild your app to see the changes.
  </Step>
</Steps>

<Tip>
The `linkedin` and `github` fields should contain just your username, not the full URL. The app handles formatting automatically.
</Tip>

## Managing skills

Skills are defined using the `Skill` struct:

```swift
struct Skill: Identifiable {
    var id: String
    let skillName: String
    let image: String  // SF Symbol name
}
```

Here's the current skills configuration from `AppModel.swift`:

skills: [

<Steps>
  <Step title="Choose an SF Symbol">
    Browse available icons at [SF Symbols](https://developer.apple.com/sf-symbols/) or use the SF Symbols app.
  </Step>

  <Step title="Add to the skills array">
    Insert a new `Skill` object:
    
    ```swift
    Skill(
        id: UUID().uuidString,
        skillName: "Swift",
        image: "swift"
    ),
    ```
  </Step>

  <Step title="Maintain consistent formatting">
    Keep skill names concise and use relevant SF Symbols that are easily recognizable.
  </Step>
</Steps>

<Warning>
Each skill must have a unique `id`. Always use `UUID().uuidString` to generate unique identifiers automatically.
</Warning>

### Removing or reordering skills

To remove a skill, simply delete its entry from the array. To reorder, cut and paste entries in your desired sequence.

## Profile photo

The profile photo is referenced in `HeaderView.swift:18`:

```swift
Image("display_photo")
    .resizable()
    .frame(width: 100, height: 100)
```

<Steps>
  <Step title="Add your image to assets">
    Add your photo to the Assets.xcassets folder in Xcode.
  </Step>

  <Step title="Name it 'display_photo'">
    Either name your asset "display_photo" or update the image name in `HeaderView.swift`.
  </Step>

  <Step title="Optimize image size">
    Use a square image at least 200x200 pixels for best quality on all devices.
  </Step>
</Steps>

<Tip>
For the best circular profile photo appearance, use a square image with important content centered, as the edges will be cropped by the circular frame.
</Tip>

## Best practices

- **Keep descriptions concise**: The description field works best with 1-2 sentences
- **Use SF Symbols**: Stick to SF Symbols for skill icons to maintain consistency
- **Order matters**: List experience and skills in reverse chronological order (most recent first)
- **Unique IDs**: Always use `UUID().uuidString` for generating unique identifiers
- **Test on device**: Preview changes on actual devices to ensure proper layout and sizing
