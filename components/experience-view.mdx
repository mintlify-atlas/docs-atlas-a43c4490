## Overview

The `ExperiencesView` displays a vertical list of work experience entries with an interactive collapse/expand feature. Users can toggle the visibility of experience details using a chevron button, similar to the SkillsView component.

<Note>
  This component uses SwiftUI's state management and animation system to create smooth transitions when showing or hiding experience entries.
</Note>

## Source code

```swift
import SwiftUI

struct ExperiencesView: View {
    
    // Variables
    var experience: [Experience]
    
    @State var showExperiences = true
    
    // Views
    var body: some View {
        VStack(alignment: .leading, spacing : 24) {
            HStack(spacing: 16) {
                Text("Experience")
                    .font(Montserrat.bold.font(size: 21))
                    .opacity(0.9)
                
                Button {
                    withAnimation(.easeInOut(duration: 0.4)) {
                        showExperiences.toggle()
                    }
                } label: {
                    Image(systemName: "chevron.up")
                        .font(.system(size: 18, weight: .medium))
                        .rotationEffect(self.showExperiences ? .zero : .degrees(180))
                }.buttonStyle(PlainButtonStyle())
            }.padding(.bottom, 14)
            
            if(showExperiences) {
                ForEach(experience) {experience in
                    ExperienceView(experience: experience)
                }
            }
        }
    }
}
```

## Properties

<ParamField path="experience" type="[Experience]" required>
  Array of Experience objects to display. Each experience entry is rendered as an individual ExperienceView component in chronological order.
</ParamField>

<ParamField path="showExperiences" type="@State Bool" default="true">
  State variable controlling the visibility of the experience list. Initialized to `true`, displaying experience entries by default.
</ParamField>

## Interactive features

### Collapsible animation

The view implements the same smooth animation pattern as SkillsView:

```swift
Button {
    withAnimation(.easeInOut(duration: 0.4)) {
        showExperiences.toggle()
    }
} label: {
    Image(systemName: "chevron.up")
        .font(.system(size: 18, weight: .medium))
        .rotationEffect(self.showExperiences ? .zero : .degrees(180))
}.buttonStyle(PlainButtonStyle())
```

<ParamField path="animation" type="easeInOut">
  - **Duration**: 0.4 seconds
  - **Type**: Ease in/out for natural motion
  - **Consistency**: Matches SkillsView animation for cohesive UX
</ParamField>

### Chevron indicator

<ParamField path="chevron.up" type="SF Symbol">
  - **Size**: 18pt with medium weight
  - **Expanded state**: Points up (0 degrees)
  - **Collapsed state**: Points down (180 degrees rotation)
  - **Purpose**: Visual indicator of collapsible state
</ParamField>

<Tip>
  The chevron rotation provides consistent interaction feedback across both the Skills and Experience sections, creating a unified user experience.
</Tip>

### Experience list

```swift
if(showExperiences) {
    ForEach(experience) {experience in
        ExperienceView(experience: experience)
    }
}
```

- **Conditional rendering**: Only renders when `showExperiences` is `true`
- **Iteration**: Uses ForEach to iterate through experience array
- **Components**: Each entry rendered as separate ExperienceView

<Note>
  The conditional rendering optimizes performance by removing experience views from the view hierarchy when collapsed, rather than just hiding them.
</Note>

## Animation behavior

When the user interacts with the toggle button:

1. **Tap detected**: Button action triggers
2. **Animation wraps state change**: `withAnimation(.easeInOut(duration: 0.4))`
3. **State toggles**: `showExperiences` flips between true/false
4. **Visual transitions**:
   - Chevron smoothly rotates 180 degrees
   - Experience list fades in/out
   - Layout height adjusts fluidly
   - Parent ScrollView adapts to content size

<Tip>
  The 0.4-second duration provides a balance between responsiveness and smoothness, allowing users to perceive the transition without feeling sluggish.
</Tip>

## Design patterns

### Consistent with SkillsView

Both sections share identical interaction patterns:

| Feature | SkillsView | ExperiencesView |
|---------|------------|------------------|
| Animation duration | 0.4s | 0.4s |
| Animation type | easeInOut | easeInOut |
| Default state | Expanded | Expanded |
| Toggle icon | Chevron | Chevron |
| Font size | 21pt | 21pt |
| Font weight | Bold | Bold |

<Tip>
  This consistency helps users quickly understand the interaction model without learning new patterns for each section.
</Tip>

### Vertical spacing hierarchy

- **24pt**: Between experience entries (main content spacing)
- **16pt**: Between title and button (header element spacing)
- **14pt**: Below header before content (section separation)
