## Overview

The `HeaderView` displays the top section of the resume, featuring a circular profile photo, name, role, location, social media links (LinkedIn and GitHub), and a professional description. This component creates the first impression and provides essential contact information.

## Source code

```swift
import SwiftUI

struct HeaderView: View {
    
    var appModel: AppModel
    
    var body: some View {
        VStack {
            HStack {
                Spacer()
                Image("display_photo")
                    .resizable()
                    .frame(width: 100, height: 100)
                    .padding(-0.94)
                    .background(
                        Circle()
                            .opacity(0.5)
                            .shadow(radius: 7)
                    )
                Spacer()
            }
            
            Text(appModel.portfolio.name)
                .font(Montserrat.bold.font(size: 17))
                .padding(.top, 8)
            
            Text(appModel.portfolio.role)
                .font(Montserrat.mediumItalic.font(size: 18))
                .opacity(0.85)
                .padding(.top, -2)
            
            HStack {
                Image(systemName: "location.fill")
                    .font(.system(size: 18, weight: .semibold))
                
                Text(appModel.portfolio.location)
                    .font(Montserrat.medium.font(size: 17))
            }
            .padding(.top, 8)
            .opacity(0.45)
            
            HStack {
                Image("linkedin")
                    .resizable()
                    .frame(width: 25, height: 25)
                
                Text(appModel.portfolio.linkedin)
                    .font(Montserrat.medium.font(size: 17))
            }
            .padding(.top, 6)
            .opacity(0.40)
            
            HStack {
                Image("github")
                    .resizable()
                    .frame(width: 25, height: 25)
                
                Text(appModel.portfolio.github)
                    .font(Montserrat.medium.font(size: 17))
            }
            .padding(.top, 6)
            .opacity(0.40)
            
            Text(appModel.portfolio.description)
                .font(Montserrat.mediumItalic.font(size: 16))
                .opacity(0.5)
                .padding(.top, 24)
                .lineSpacing(10)
        }
    }
}
```

## Properties

<ParamField path="appModel" type="AppModel" required>
  The app data model containing portfolio information including:
  - `name`: Full name of the portfolio owner
  - `role`: Professional title or role
  - `location`: Geographic location
  - `linkedin`: LinkedIn profile handle
  - `github`: GitHub username
  - `description`: Professional bio or summary
</ParamField>

## Component sections

### Profile photo

```swift
Image("display_photo")
    .resizable()
    .frame(width: 100, height: 100)
    .padding(-0.94)
    .background(
        Circle()
            .opacity(0.5)
            .shadow(radius: 7)
    )
```

- **Size**: 100x100 points
- **Shape**: Circular with shadow backdrop
- **Shadow radius**: 7pt for depth effect
- **Alignment**: Horizontally centered using spacers

<Tip>
  The negative padding (-0.94) fine-tunes the image positioning within the circular background.
</Tip>

<ParamField path="role" type="Text">

## Typography

The HeaderView uses the Montserrat font family throughout:

- **Bold**: For the name (primary identifier)
- **Medium Italic**: For role and description (professional context)
- **Medium**: For location and social links (supporting info)

## Visual hierarchy

Opacity levels create a clear visual hierarchy:

1. **Name** (100%): Primary focus
2. **Role** (85%): Secondary identifier
3. **Description** (50%): Tertiary supporting text
4. **Location** (45%): Supporting detail
5. **Social links** (40%): Supplementary information

<Tip>
  The opacity gradient guides the reader's eye from most important (name) to supporting details (links), creating a natural reading flow.
</Tip>
